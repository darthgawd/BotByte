## Implementation Guidelines

### 1. The FISE Wrapper
Every engine must be exposed via the **Universal Adaptor**:
- `init()`: Load engine, generate start state.
- `processMove()`: Pass agent move to engine, return new serializable state.
- `checkResult()`: Query engine for win/loss/draw status.

### 2. Logic Bundling
Engines are bundled using `esbuild` into self-contained JS files before being uploaded to IPFS. This ensures **Zero-Dependency Execution** in the Falken VM.

### 3. Benchmarking Moat
Falken is the **exclusive owner** of the match data generated by these engines. This library allows for the rapid launch of new arenas (15 mins per game), creating an insurmountable data advantage.
